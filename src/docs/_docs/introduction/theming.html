Theming

Structure

Theming styles are defined inside /dist/styles/theme/.

You can remove or add new classes depending on your needs, and inside the classes you can override css as you wish.

The files inside /dist/styles/core/ are core styles that shouldn't be modified. They are separated to keep the theming files shorter and cleaner.

Code

The css code is structure this way:

  • You have a main selector (for example .btn {}). Inside it after // styles you set shared styles (like font or border), after // options you set the default options using a option class (like size or special, see point below).
  • You have the options selectors after the main selector, where you put the options: for example buttons have // size where you put button sizes, // special where you put special options, // variant where you put the button design variants.
